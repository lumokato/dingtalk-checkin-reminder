# 打卡提醒

面向钉钉极速打卡场景的 Android 提醒器。它只负责在合适的时间、地点和手机解锁状态下提醒并拉起钉钉，不伪造打卡、不使用无障碍、不截图、不模拟点击。

建议仓库名：`dingtalk-checkin-reminder`

## 功能

- 上班窗口内启动前台服务和定位检测。
- 进入配置范围且手机解锁后，尝试拉起钉钉触发极速打卡。
- 下班按规则页配置的正常下班时间提醒；如果已记录上班且上班晚于正常上班时间，则下班时间顺延对应迟到时长。
- 支持高德地图分享链接/短链导入地点，也支持手动填写 GCJ-02 坐标和半径。
- 支持通知监听识别钉钉打卡成功通知；识别失败时仍保留手动确认入口。
- 上班窗口结束仍未确认时，记录为上班未确认，不阻塞当天正常下班提醒。
- 日志保存在应用私有目录，便于排查定位、解锁、拉起钉钉等问题。

## 隐私

- 源码不内置个人地点、公司名、坐标或高德短链。
- 用户配置的地点、时间和状态只保存在本机应用数据中。
- 应用不上传定位、不上传日志、不接入第三方统计。
- 公开仓库不要提交 `local.properties`、签名文件、构建产物、设计临时文件或个人截图；这些已在 `.gitignore` 中排除。

## 权限

应用会按功能申请以下权限：

- 精确定位和后台定位：用于在上班窗口内判断是否进入打卡范围。
- 前台服务：用于在检测窗口内保持后台检测。
- 通知：用于提醒和手动确认入口。
- 精确闹钟：用于按上班/下班窗口唤起检测。
- 通知监听服务：可选，用于识别钉钉打卡成功通知。

不同 Android 厂商系统可能仍会限制后台定位、前台服务或后台拉起应用。建议把本应用设为电池“不受限制”，并在最近任务里锁定。

## 本地构建

本项目不使用 GitHub Actions 构建 APK。发布 APK 必须在可信本机环境构建并上传到 GitHub Release。

```powershell
.\gradlew.bat assembleRelease
```

APK 输出：

```text
app\build\outputs\apk\release\app-release.apk
```

如果需要使用私有签名文件，可以在本机环境变量中指定：

```powershell
$env:DINGTALK_REMINDER_SIGNING_PROPERTIES="C:\path\to\signing.properties"
.\gradlew.bat assembleRelease
```

`signing.properties` 格式：

```properties
storeFile=C:/path/to/release.jks
storePassword=...
keyAlias=...
keyPassword=...
```

## 首次使用

1. 安装 Release 页面下载的 APK。
2. 打开应用，配置地点、范围、上班时间、允许迟到时间和下班时间。
3. 申请定位、通知、精确闹钟等必要权限。
4. 按需开启通知监听，用于识别钉钉打卡成功通知。
5. 在系统电池设置中允许后台运行。
6. 开启守护。

## 当前限制

- 钉钉极速打卡是否成功由钉钉自身决定，本应用不绕过钉钉规则。
- Android/厂商系统可能限制后台直接打开钉钉；这种情况下依赖通知和手动入口兜底。
- 通知监听识别依赖钉钉通知内容，钉钉改版后可能需要更新匹配逻辑。
